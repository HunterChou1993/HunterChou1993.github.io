                                      # 12 生命周期

                                      # React生命周期

                                      生命周期分为三个阶段

                                      • 挂载时
                                        • constructor -> render -> componentDidMount
                                      • 更新时
                                      • 卸载时

                                      # 常用生命周期

                                      # constructor

                                      • 构造方法先执行
                                      • 如果步初始化state或不进行方法绑定，则不需要为React组件实现构造函数
                                      • constructor 通常制作两件事情
                                        • 通过给this.state赋值对象来初始化内容的state
                                        • 为事件绑定实例

                                      # componentDidMount

                                      • 会在组件挂载后立即调用
                                      • 相关操作
                                        • 依赖于DOM的操作
                                        • 发送网络请求
                                        • 添加订阅(但是要在componentWillUnmount取消)

                                      # componentDidUpdate

                                      • 会在更新后立即被调用，首次渲染不执行该方法
                                      • 相关操作
                                        • 前后props进行比较，可以在这里发送网络请求

                                      # componentWillUnmount

                                      • 会在组件卸载及销毁之前直接调用
                                      • 相关操作
                                        • 清除timer ,取消网络请求或者清除订阅

                                      # 不常用生命周期

                                      • getDerivedStateFromProps
                                        • state的值在任何时候都依赖于props时使用，该方法返回一个对象来更新state
                                      • getSnapshotBeforeUpdate
                                        • 在React更新DOM之前回调一个函数，可以获取DOM更新钱的一些信息(比如说滚动位置)
                                      • shouldComponentUpdate
                                        • 性能优化的时候用到
